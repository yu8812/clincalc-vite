📖 連線教學（5分鐘完成）
1
建立 Supabase 帳號與專案
前往 supabase.com 免費註冊。點選「New project」，填入專案名稱（如 clincalc）、資料庫密碼（請妥善保存），選擇亞太伺服器（如 Singapore）。等待約 1-2 分鐘建立完成。
2
建立資料表（SQL Editor）
在左側選單點選「SQL Editor」→「New query」，貼入下方 SQL 並執行：
-- 建立受檢者資料表 CREATE TABLE patients ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, created_at timestamptz DEFAULT now(), updated_at timestamptz DEFAULT now(), doctor_id text NOT NULL, pid text NOT NULL, exam_date text, sex text, age integer, data jsonb, qualify_result jsonb, ai_summary jsonb, notes text ); -- 建立索引加速查詢 CREATE INDEX ON patients (pid); CREATE INDEX ON patients (doctor_id); CREATE INDEX ON patients (exam_date); -- 啟用 Row Level Security ALTER TABLE patients ENABLE ROW LEVEL SECURITY; -- 建立存取政策（anon key 可讀寫） CREATE POLICY "allow_all" ON patients FOR ALL USING (true);
點選「Run」執行，出現「Success」即完成。
3
取得 API 連線資訊
在 Supabase 側欄點選「Settings」→「API」：
• Project URL：複製「Project URL」（格式：https://xxxxx.supabase.co）
• Anon Key：複製「Project API keys」→「anon public」
⚠️ 請勿將 service_role key 貼入（有完全管理權限，請妥善保管）
4
填入連線資訊並測試
將 URL 和 Key 貼入下方欄位，點選「儲存」後點選「測試連線」，出現綠色成功訊息即完成。
連線設定
anon key 為前端安全 key，不具管理員權限。
常見問題
❓ 「連線失敗 404」：資料表尚未建立，請先執行步驟 2 的 SQL。
❓ 「CORS 錯誤」：請確認 Project URL 格式正確，不含尾巴斜線。
❓ 「401 Unauthorized」：請確認貼入的是 anon key，而非 service_role key。
❓ 資料安全：Supabase 提供 Row Level Security（RLS），可進一步設定只有特定醫師能存取自己的資料。
🔒 去識別化模式已啟用：系統自動移除所有可識別個人身份的欄位（姓名、身分證號、電話、地址等），僅保留必要的數值與檢驗結果。
